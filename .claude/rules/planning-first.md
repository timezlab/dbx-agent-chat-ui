# Planning First

For any multi-step, multi-file, or architecturally-novel task: **plan before coding, then execute with discipline.** Skip only for trivial single-file work (≤30 min, obvious shape).

- **Explore first, then plan.** Never plan against an unverified mental model.
- **Hard gate:** do not write implementation code until the user approves the plan.
- **Verify after every task.** A failing check is a blocker — stop and escalate, don't force through.
- **If reality diverges, update the plan before coding further.**

See `.agents/skills/planning-first/` for clarifying-question discipline, trade-off framing, and the full execution protocol.
